Question-2: What is the difference between innerHTML, innerText, and textContent ?

innerHTML: accesses or modifies an element's HTML content, including tags.
innerText: deals with visible, styled text, excluding hidden elements.
textContent: retrieves or sets all text content, ignoring styling, making it faster and suitable for raw text manipulation.

Question-3: What is event delegation in the DOM?

Event delegation in the DOM is a technique where a single event listener is attached to a parent element to manage events from multiple child elements. This method leverages event bubbling, where events initiated by child elements propagate up to their parent, allowing for centralized event handling. It reduces the number of event listeners needed, improves performance, and simplifies management of dynamic content. By using event delegation, applications can handle events for newly added elements automatically without the need to attach specific listeners to each child.

Question-4: What is event bubbling in the DOM?

Event bubbling in the DOM is a process where an event initiated on a specific element propagates upwards through its parent elements. This mechanism allows a single event handler on a higher-level parent to respond to events triggered on its child elements, facilitating simplified and efficient event management, especially in dynamic interfaces. This behavior is foundational for implementing event delegation, where events are handled at a communal ancestor rather than at individual descendant elements.
